# Project Structure

Annotated file tree. Files are grouped by architectural role, not alphabetically. Read order for a new developer is from top to bottom — this list is roughly "most important" to "least important".

```
cue/
├── project.yml                        # xcodegen config: macOS target, entitlements, build settings
├── Cue/
│   ├── CueApp.swift                   # @main entry point. Hosts Settings scene + About panel.
│   ├── AppDelegate.swift              # NSPanel setup, menu bar item, global key monitor.
│   ├── ContentView.swift              # Main UI + session lifecycle coordinator. Owns all services.
│   ├── Services/
│   │   ├── MicLevelMonitor.swift      # AVCaptureSession mic capture. Forwards CMSampleBuffers.
│   │   ├── SpeechTranscriber.swift    # SFSpeechRecognizer wrapper. Accepts CMSampleBuffer.
│   │   ├── TranscriptionMatcher.swift # Fuzzy rolling-lookahead matcher. Script → position.
│   │   └── Logger.swift               # DEBUG-only file logger (~/Library/Logs/Cue/app.log). No-op in Release.
│   ├── Views/
│   │   ├── TeleprompterView.swift     # NSTextView-backed scroll surface. Scrolls to char offset.
│   │   ├── CountdownView.swift        # 3-2-1 overlay shown before speech starts.
│   │   └── SettingsView.swift         # Settings form: opacity, text size defaults.
│   ├── Info.plist                     # Generated by xcodegen (INFOPLIST_KEY_* in project.yml)
│   └── Cue.entitlements               # Sandbox ON, mic + speech entitlements
├── Cue.xcodeproj/                     # Generated by xcodegen — not hand-edited
├── plan.md                            # Current state + next steps + historical v0 spec
├── plan-dumb.md                       # Alternative simpler spec (kept for reference)
├── backlog.md                         # Open work: testing, v2 features, housekeeping
├── CHANGELOG.md                       # Shipped features by session
├── BUILD_REPORT.md                    # Output from the initial /v0-mac ralph-loop build
├── README.md                          # Project overview
├── wiki/                              # This wiki (you are here)
├── checkpoint.json                    # Phase state from the initial autonomous build
└── .v0-mac/
    ├── prompt.md                      # Full prompt fed to ralph-loop during initial build
    └── screenshots/                   # Visual QA screenshots from Phase 4
```

## Recently removed

During the 2026-04-20 cleanup pass, two dead files were deleted: `ScrollController.swift` (timer-based scroll from the pre-transcription era) and `VolumeMeterView.swift` (old mic-toggle diagnostic). Both were compiled but unreferenced. Restore from git history if a future feature needs them.

## Key Files to Read First

For understanding the app in order of importance:

1. **ContentView.swift** — the whole app from the user's perspective in ~240 lines. Controls, state machine, service wiring.
2. **MicLevelMonitor.swift** — audio capture. Non-obvious because it uses `AVCaptureSession` instead of the usual `AVAudioEngine` — see [Design-Decisions.md](Design-Decisions.md).
3. **TranscriptionMatcher.swift** — the fuzzy matcher. The most algorithmically interesting file. All the robustness against misrecognition and off-script talk lives here.
4. **TeleprompterView.swift** — SwiftUI/AppKit bridge. Wraps `NSTextView` so we can query glyph positions and scroll to an exact character.
5. **AppDelegate.swift** — why the window is floating, notch-shaped, invisible to screen capture, and how keyboard shortcuts work globally within the app.

## Generated Files (don't hand-edit)

- `Cue.xcodeproj/` — regenerated by `xcodegen generate` from `project.yml`
- `Info.plist` — synthesized by Xcode from `INFOPLIST_KEY_*` build settings in `project.yml`
- Anything under `~/Library/Developer/Xcode/DerivedData/Cue-*/`
